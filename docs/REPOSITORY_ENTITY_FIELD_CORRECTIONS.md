### 1. UserRepositoryCustomImpl ✅

**Before (WRONG):**
```java

    User user = new User();
    user.setUserId(rs.getLong("user_id"));
    user.setEmailAddress(rs.getString("email_address"));
```

**After (CORRECT):**
```java
private final RowMapper<User> userRowMapper = (rs, rowNum) -> {
    User user = new User();
    user.setUserId(rs.getLong("user_id"));
    user.setEmailAddress(rs.getString("email_address"));
    user.setPassword(rs.getString("password"));
    
    // Map status enum ✅
    String statusStr = rs.getString("status");
    if (statusStr != null) {
        user.setStatus(UserStatus.valueOf(statusStr));
    }
    
    // Map timestamp ✅
    Timestamp timestamp = rs.getTimestamp("timestamp");
    if (timestamp != null) {
        user.setTimestamp(timestamp.toLocalDateTime());
    }
    
    // Map failed login attempts ✅
    Integer failedAttempts = rs.getInt("failed_login_attempts");
    user.setFailedLoginAttempts(failedAttempts != null ? failedAttempts : 0);
    
    // Map account locked until ✅
    Timestamp lockedUntil = rs.getTimestamp("account_locked_until");
    if (lockedUntil != null) {
        user.setAccountLockedUntil(lockedUntil.toLocalDateTime());
    }
    
    return user;
};
```

## Key Learnings

### 1. Always Reference the Entity ✅

Before writing RowMapper code, always check:
```java
// Look at the actual entity class
@Entity
@Table(name = "table_name")
public class EntityName {
    // Check actual field names
    // Check data types
    // Check annotations (@Id, @Column, etc.)
}
```

### 2. String vs Long Primary Keys ✅

Both ParticipantProfile and OrganizerProfile use **String** primary keys with auto-generated codes:
- ParticipantProfile: `P1`, `P2`, `P3`...
- OrganizerProfile: `O1`, `O2`, `O3`...

These are generated by custom ID generators:
- `StringSequenceIdGenerator` for ParticipantProfile
- `OrganizerIdGenerator` for OrganizerProfile

### 3. Computed Columns ✅

Some fields are computed by the database:
- ParticipantProfile.age - Calculated from date_of_birth
- Marked as `insertable = false, updatable = false`

These should be mapped but not included in INSERT/UPDATE statements.

### 4. Enum Fields ✅

Enums should be mapped carefully:
```java
// Read enum as String from DB
String statusStr = rs.getString("status");
if (statusStr != null) {
    // Convert to enum
    entity.setStatus(EnumType.valueOf(statusStr));
}
```

### 5. Null Handling ✅

Always check for nulls:
```java
// For Timestamp
Timestamp ts = rs.getTimestamp("column_name");
if (ts != null) {
    entity.setField(ts.toLocalDateTime());
}

// For Date
Date date = rs.getDate("column_name");
if (date != null) {
    entity.setField(date.toLocalDate());
}

// For primitives that might be null
Integer value = rs.getInt("column_name");
if (!rs.wasNull()) {
    entity.setField(value);
}
```

## Database Table Structures

### User Table
```sql
CREATE TABLE [user] (
    user_id BIGINT PRIMARY KEY IDENTITY,
    email_address NVARCHAR(255) UNIQUE NOT NULL,
    password NVARCHAR(255) NOT NULL,
    status NVARCHAR(50) NOT NULL DEFAULT 'AUTHENTICATED',
    timestamp DATETIME,
    failed_login_attempts INT DEFAULT 0,
    account_locked_until DATETIME
);
```

### Participant Profile Table
```sql
CREATE TABLE participant_profile (
    profile_code NVARCHAR(20) PRIMARY KEY, -- P1, P2, P3...
    user_id BIGINT UNIQUE NOT NULL,
    full_name NVARCHAR(255),
    phone_number NVARCHAR(20),
    date_of_birth DATE,
    age AS DATEDIFF(YEAR, date_of_birth, GETDATE()), -- Computed
    FOREIGN KEY (user_id) REFERENCES [user](user_id)
);
```

### Organizer Profile Table
```sql
CREATE TABLE organizer_profile (
    profile_code NVARCHAR(20) PRIMARY KEY, -- O1, O2, O3...
    user_id BIGINT UNIQUE NOT NULL,
    official_name NVARCHAR(255),
    tax_id NVARCHAR(50),
    logo_url NVARCHAR(500),
    status NVARCHAR(50) DEFAULT 'PENDING',
    FOREIGN KEY (user_id) REFERENCES [user](user_id)
);
```

### User Role Table
```sql
CREATE TABLE user_role (
    user_id BIGINT NOT NULL,
    role NVARCHAR(50) NOT NULL,            -- Column is 'role' not 'arole'
    PRIMARY KEY (user_id, role),           -- Composite PK
    FOREIGN KEY (user_id) REFERENCES [user](user_id),
    CONSTRAINT chk_user_role CHECK (role IN ('ROLE_PARTICIPANT', 'ROLE_ORGANIZER', 'ROLE_ADMIN'))
);

-- Note: No assigned_at or timestamp column exists
```
